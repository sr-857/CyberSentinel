CyberSentinel

CyberSentinel is a threat-intelligence ingestion and log-correlation pipeline built for SOC analysts. It captures IOCs, parses SSH and web logs, scores correlations, raises alerts, and exposes analytics through a single pane of glass.

Launch Live Demo View Source on GitHub Release Notes

What CyberSentinel Does

  • Ingests IOCs from AbuseIPDB and AlienVault OTX, normalises indicators, and persists confidence metadata.
  • Parses SSH authentication and Apache access logs to capture IPs, users, HTTP verbs, status codes, and timestamps.
  • Correlates parsed events with live intel to produce severity-weighted alerts.
  • Builds analyst dashboards with KPI tiles, trend charts, and data tables powered by Chart.js.
  • Offers a workflow refresh endpoint to chain intel ingestion, log parsing, correlation, and analytics in a single call.

Why It Matters

CyberSentinel demonstrates pragmatic security engineering. It automates the grind of gathering intel, cleansing telemetry, and prioritising threats so analysts can focus on response. Recruiters and hiring managers see how detection logic, data pipelines, and UX fit together.

Architecture Overview

The platform is composed of modular services backed by a lightweight SQLite store and exposed through Flask APIs.

  • Threat Feeds: AbuseIPDB and AlienVault OTX collectors with caching.
  • Log Parsers: SSH and Apache pipelines with structured field extraction.
  • Correlation Engine: Severity scoring based on IOC matches with contextual metadata.
  • Analytics Builder: KPI aggregation and Chart.js-ready datasets.
  • Dashboard: Static frontend consuming REST endpoints for intel, logs, alerts, and analytics.

Get Started

Deploy CyberSentinel quickly with Docker or run the backend locally for development.

# Docker Compose
docker compose up --build

# Local development
python -m venv .venv
source .venv/bin/activate
pip install -r backend/requirements.txt
python -m backend.db         # initialise SQLite schema
python -m backend.app        # launch Flask API

Open the dashboard and click Fetch Intel → Parse Logs → Run Correlation → Refresh Workflow to watch the full pipeline execute end-to-end.
